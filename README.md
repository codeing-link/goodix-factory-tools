# GH Protocol Web 测试使用手册（客户版）

## 1. 文档目的
本文档用于指导客户基于当前 Web 工具完成完整测试流程。按照本文档操作，可完成：
- 设备连接与模式设置
- 配置文件导入（手动/批量自动）
- 自动采样并生成 CSV
- LPCTR/LPLCTR/BaseNoise/PPGNoise 结果判定

## 2. 测试前准备

### 2.1 必备文件
请准备同一目录下的 4 组配置文件（文件名前缀）：
- `Base*`
- `LPCTR*`
- `PPG*`
- `LPLCTR*`

示例：
- `Base_Noise_TEST1_100Hz_0327.config`
- `LPCTR_TEST1_100Hz_0327.config`
- `PPG_Noise_TEST1_100Hz_0327.config`
- `LPLCTR_TEST1_100Hz_0327.config`

说明：
- 系统会自动忽略隐藏文件和系统文件（如 `.DS_Store`、`Thumbs.db`）。
- 批量自动测试按上述前缀自动匹配并执行。

### 2.2 后端启动
在后端目录启动程序：

Windows:
```bat
gh_backend.exe --port COM28 --baud 115200
```

macOS:
```bash
./gh_backend --port /dev/tty.usbserial-310 --baud 115200
```

打开浏览器访问：
- `http://localhost:8080`
- 也可以直接打开本地的index.html离线网页

## 3. 测试环境要求

### 3.1 上传配置前（批量测试按钮前置确认）
上传前，请确认如下测试环境：
1. 黑暗环境，无白炽光干扰，无环境光干扰。  
（验证阶段：可用黑色隔光泡棉进行遮盖）
2. 白色亚克力块盖住光窗，并保持静止。
3. 制作成治具后，白色亚克力和光窗间隔 1mm；各个治具的间隔统一。

### 3.2 最后一项 LPLCTR 前（流程中二次确认）
更换测试环境，环境要求：
1. 黑暗环境，无白炽光干扰，无环境光干扰。  
（验证阶段：可用黑色隔光泡棉对纸盒内壁包裹）  
（验证阶段：在无光源的会议室，关闭所有窗和灯光）
2. 光窗对空，保持静止。若使用黑盒，需要保持净空 50cm。
3. 双色注塑方案的设备，可以使用黑色橡皮泥进行遮挡光窗；盖板方案只能黑暗环境对空进行测试。

## 4. Web 页面操作说明

### 4.1 连接设备
1. 选择串口和波特率（默认 `115200`）。
2. 点击“连接设备”。
3. 状态显示“已连接”后继续。

### 4.2 模式选择与按钮可用性
“模式选择”有两个模式：
- `Mcu Online Work Mode`
- `Auto Pass Through Mode`

界面联动规则：
- 在 `Mcu Online` 模式：
  - “🤖 批量自动测试”禁用（灰色不可点）。
- 在 `Auto Pass Through` 模式：
  - “📁 导入配置”禁用（灰色不可点）。
  - “Test1”、“开始采集”、“停止采集”禁用（灰色不可点）。

## 5. 推荐测试流程（批量自动）

1. 切换到允许批量的模式（按当前界面规则）。
2. 点击“🤖 批量自动测试”。
3. 选择包含 4 组配置文件的目录。
4. 系统自动按顺序执行：
   - `Base*`：自动采样 200 条并保存 CSV
   - `LPCTR*`：自动采样 200 条并保存 CSV
   - `PPG*`：自动采样 200 条并保存 CSV
   - `LPLCTR*`：弹窗提醒更换环境，确认后自动采样 200 条并保存 CSV
5. 测试完成后，在结果区查看 LED 判定和数值。

说明：
- 每个配置默认采样目标为 `200` 条。
- 超时会自动中断并尝试自动发送停止采样。

## 6. 手动流程（可选）
如需单文件调试：
1. 点击“📁 导入配置”，导入单个 `.config`。
2. 点击“发送”下发寄存器。
3. 点击“开始采集/停止采集”完成采样。
4. 查看同名 CSV 输出和结果区。

寄存器表显示优化：
- 默认仅显示前 5 行。
- 点击“展开全部”可查看全部寄存器。

## 7. 输出文件说明
每次开始采样时，系统会按配置名自动生成同名 `.csv` 文件。

示例：
- `Base_Noise_TEST1_100Hz_0327.csv`：用于 BaseNoise 计算
- `LPCTR_TEST1_100Hz_0327.csv`：用于 LPCTR 计算
- `PPG_Noise_TEST1_100Hz_0327.csv`：用于 PPGNoise 计算
- `LPLCTR_TEST1_100Hz_0327.csv`：用于 LPLCTR 计算

CSV 主要包含：
- 时间戳、帧号、ACC、CH0~CH15、FLAG、REF_RESULT、ALGO_RESULT、AGC_INFO 等列

## 8. 指标计算原理

## 8.1 LPCTR / LPLCTR（单位nA）
数据来源：对应 CSV 的 `CH0..CH15`，前一半通道是实际采样的ADC原始值（采样精度24bit，满量程是1.8V），后一半通道是经过转换的电流值pA。

计算步骤：
1. 取最新 `100` 行数据窗口。
2. 统计非 0 通道数 `N`（在 CH0..CH15 中，窗口内出现过非 0 即计入）。
3. 取后半段通道：`[N/2, N-1]`。
4. 每个通道计算窗口平均值 `avg`（pA）。
5. 结果值：`avg / 24（led电流，单位nA/mA）/ 1000`。

输出：
- 每个通道一个值，对应一个 LED 卡片显示。

## 8.2 BaseNoise / PPGNoise（单位uV）
数据来源：对应 CSV 的 `CH0..CH15`，前一半通道是实际采样的ADC原始值（采样精度24bit，满量程是1.8V），后一半通道是经过转换的电流值pA。

计算步骤：
1. 取最新 `100` 行数据窗口。
2. 统计非 0 通道数 `N`。
3. 取前半段通道：`[0, N/2-1]`。
4. 每个通道执行 0.5Hz 高通滤波（`fs=100Hz`，7 阶，系数与原 C 逻辑一致）。
5. 对滤波后序列计算标准差 `std_rawdata`。
6. 电压换算：`noise_uv = std_rawdata / (2^24) * 1.8 * 1000000uV`。

输出：
- 每个通道一个 noise 值（`values`）。
- 同时保留通道平均值用于汇总（界面主显示按通道灯展示）。

## 9. 判灯规则与默认阈值

默认阈值：
- `LPCTR = 24`
- `LPLCTR = 0.8`
- `BaseNoise = 7`
- `PPGNoise = 280`

绿灯/红灯判定：
- `LPCTR`：`值 > 阈值` 为绿色，否则红色
- `LPLCTR`：`值 < 阈值` 为绿色，否则红色
- `BaseNoise`：`值 < 阈值` 为绿色，否则红色
- `PPGNoise`：`值 < 阈值` 为绿色，否则红色

## 10. 常见问题排查

### 10.1 批量流程超时（未到 200 条）
现象：
- 日志显示 `采样超时: xxx 200条`

建议排查：
1. 先确认设备是否仍在持续上报（串口调试区是否有 RX）。
2. 检查光学环境是否满足当前步骤要求。
3. 重新执行该步骤；如仍失败，断电重启下位机后重测。

### 10.2 目录明明 4 个文件却显示更多
原因：
- 目录存在隐藏文件或系统文件（如 `.DS_Store`）。
说明：
- 系统已做过滤，实际匹配仍按 `Base/LPCTR/PPG/LPLCTR` 四类文件。

### 10.3 模式下按钮灰色不可点
这是预期行为，用于防误操作。请根据当前模式切换到允许该操作的模式。
