# GH Protocol 重构版本说明（2026-03-30）

本文档记录当前可用版本在「串口连接 → 配置下发 → 启动采集 → 数据上报/CSV」链路上的关键修复点与验收方法。

---

## 1. 关键修复项

### 1.1 启动采集链路修复
- 对齐 Cardiff RPC `com_id` 序号窗口，避免与原上位机行为偏差导致的采集异常。
- `POST /api/device/start` 增加明确失败返回，不再“假成功”：
  - `no_service`
  - `not_connected`
  - `start_cmd_send_failed`
- 增加控制日志，便于定位启动问题：
  - `[CTRL] start req ...`
  - `[CTRL] start command sent`

### 1.2 Chelsea A 解码稳定性修复
- 修复差分解码历史值在每包重置的问题，避免采集后数据流异常或 CSV 记录异常。

### 1.3 串口默认配置统一
- 全局默认波特率统一为 `115200`（后端、API 默认值、前端默认选项、文档示例）。

### 1.4 POSIX 串口速率兼容
- 保留高波特率支持路径：
  - Linux：`B400000`（若平台支持）
  - macOS：`IOSSIOSPEED` 自定义速率
- 默认仍为 `115200`，避免设备/驱动不一致带来的大包接收异常。

---

## 2. 推荐使用流程

1. 连接设备（确认端口与波特率）。
2. 读取寄存器做链路探活（可选但推荐）。
3. 下发配置文件（`/api/device/config`）。
4. 点击开始采样（`/api/device/start`）。
5. 观察串口调试窗口与 CSV 文件是否持续更新。

---

## 3. 验收日志要点

### 3.1 启动命令与 ACK
- 可看到 `GH3X_SwFunctionCmd` 的 TX 与对应 ACK：
  - `ACK key="GH3X_SwFunctionCmd" status=0x02`

### 3.2 采集数据上报
- 启动后应出现持续 `RX` 数据流（包含大包）。
- 同时可见 CSV 开始写入日志：
  - `[CSV] Saving to <name>.csv`

### 3.3 控制层日志
- 启动请求时应出现：
  - `[CTRL] start req ...`
  - `[CTRL] start command sent`

---

## 4. 已知注意事项

- 若仅有 ACK、没有后续大包流，优先检查：
  1. 设备固件工作模式与配置文件是否匹配；
  2. 串口参数是否与设备一致；
  3. 是否完整执行了配置下发流程后再启动采样。
- 若前端显示“启动成功”但无 TX/RX，查看接口返回 `code/msg` 与 `[CTRL]` 日志定位失败点。
